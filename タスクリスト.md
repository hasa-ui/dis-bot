# タスクリスト

## 概要

この文書は `ロードマップ.md` を実行用タスクへ落とし込んだものである。今回の実行対象は短期ロードマップとし、優先順は「運用事故の予防」「日常作業の削減」「運用追跡性の強化」に合わせて整理する。

## 既存公開 I/F

- 維持する既存コマンド: `/setup` / `/status_config` / `/status_set` / `/status_clear` / `/status_view`
- 短期追加候補: `/status_list` / `/status_history member` / `/status_notify_config`
- UI 拡張で扱うもの: `/setup` 保存前プレビュー

## 実行タスク

- [ ] `/setup` 保存前プレビューを追加する。目的: 設定変更事故の予防。主な変更箇所: `status_bot/views.py` / `status_bot/service.py` / `status_bot/formatters.py`。完了条件: 保存直前に再適用対象人数・丸め対象人数・見つからないロール件数を再計算して表示し、段階数変更とロール差し替え時の想定入出力を説明できる。
- [ ] `/status_list` を追加する。目的: active record の日常確認を即時化。主な変更箇所: `status_bot/commands.py` / `status_bot/store.py` / `status_bot/formatters.py`。完了条件: guild 単位で active な `status_records` を取得し、少なくとも段階別または期限順で一覧表示でき、件数制限またはページング方針が文書化されている。
- [ ] 履歴・監査ログを追加する。目的: 手動操作・自動遷移・設定変更の追跡。主な変更箇所: `status_bot/store.py` / `status_bot/service.py` / `status_bot/commands.py`。完了条件: active record と分離した履歴保存先を持ち、`/status_history member` で主要イベントを参照でき、既存 DB と legacy migration への影響を説明できる。
- [ ] 通知設定を追加する。目的: 自動遷移や解除の見落とし防止。主な変更箇所: `status_bot/store.py` / `status_bot/service.py` / `status_bot/commands.py`。完了条件: guild ごとに通知先チャンネルと通知対象イベントを保持し、手動付与・手動解除・自動遷移・`hold` 移行・設定変更の通知可否を制御できる。

## 検証観点

- [ ] `python -m py_compile bot.py status_bot/*.py tests/*.py`
- [ ] `python -m unittest discover -s tests`
- [ ] slash command 追加項目ごとに想定入力と想定出力を整理する。
- [ ] DB 変更を伴う項目では、`guild_status_settings` / `guild_status_stages` / `status_records` と legacy migration への影響を説明する。
- [ ] `/setup` 保存前プレビューでは、段階数縮小・ロール差し替え・欠損ロール・active record あり/なしを確認する。
- [ ] `/status_list` では、0 件・複数件・`hold`・表示件数上限時の挙動を確認する。
- [ ] 履歴・通知では、手動付与・手動解除・自動遷移・`hold`・設定変更を確認する。
